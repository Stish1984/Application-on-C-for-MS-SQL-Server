Для использования приложения необходимо произвести настройки:

1. Откройте папку "DataBase" и переместите содержимое 
( два файла базы данных с расширениями .mdf и .ldf)
в директорию, где хранятся файлы Ваших баз данных.

2. Запустите MS SQL Server и присоедените скопированную базу данных.
Для этого нажмите правой кнопкой мыши на папке "Базы данных"("Database") и выберите
"Присоеденить"("Attach"). В открывшемся окне нажмите "Добавить"("Add") и укажите путь
 к скопированной базе данных. Нажмите OK. База появится в списке.

3. Остальное содержимое архива (всё, кроме папки "DataBase" ) переместите в папку с 
программами. 

4. Для работы приложения необходимо подключить его к Вашему серверу. Для этого с помощью блокнота
(либо другого текстового редактора, либо с помощью Visual Studio) откройте файлы конфигурации
Sales_registration.exe.config и App.config. Найдите строку:

<add name="ConnectionString" connectionString="data source=MYDESKTOP\SQLEXPRESS;
initial catalog=&quot;Sales registration&quot;;integrated security=True;
MultipleActiveResultSets=True;App=EntityFramework" providerName="System.Data.SqlClient" />
 
Измените параметры подключения в соответствии с настройками Вашего сервера.

Чтобы подключиться к именованному экземпляру SQL Server,используйте 
имя_сервера\имя_экземпляра(в данном случае Вам необходимо заменить "MYDESKTOP\SQLEXPRESS"
после выражения "data source=" на имена своих сервера\экземпляра).

Для других вариантов подключения рекомендуется ознакомиться с документацией по ссылке:
https://docs.microsoft.com/ru-ru/dotnet/framework/data/adonet/connection-string-syntax

5. Переместите ярлык "Sales_registration.exe" в ту директория, откуда будет запускаться приложение
 и запустите его нажав на ярлык.

Имеющаяся в базе-образце информация:

1. При запуске приложения открывается окно авторизации. Из трех аккаунтов, "SIDOROV" обладает
правами администратора, что позволяет открывать форму доступа к базе данных пользователей.
 Пароли для входа у всех соответствуют логинам только с нижним регистром.
Например:(Логин: IVANOV Пароль: ivanov).

2. Ключ шифрования для паролей по алгоритму RSA находится в строке "Key" в классе "Crypter.cs".

3. Пути при создании:
-к файлам базы данных:
 C:\Program Files\Microsoft SQL Server\MSSQL14.SQLEXPRESS\MSSQL\DATA

-к файлам приложения:
C:\Users\stesh\source\repos
